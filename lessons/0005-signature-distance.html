Faunaform · เรียนรู้เรื่องเสียง

บทเรียนที่ 0005 · ต่อจาก บทที่ 4

บทรวบยอด: ทำให้ตัวเลข Δ ตรงกับสิ่งที่หูได้ยิน

ถึงจุดที่ทุกอย่างมาบรรจบ คุณรู้แล้วว่า signature คือ สเปกตรัมเฉลี่ย (บท 3) ที่ย่อเป็น 96 แท่ง และ 4 features (บท 4) ตอนนี้คำถามสุดท้ายของ mission: เราจะวัด “ความต่าง” ของสองเสียงให้ตรงกับหูได้ยังไง? นั่นคือหัวใจของการทำให้เสียงสัตว์ ต่างชนิด “ดูแยกกันได้”

1 · วันนี้ Faunaform วัดความต่างยังไง

ฟังก์ชัน barsDistance(a, b) ใน transforms.js ทำสิ่งเดียว: หาค่า เฉลี่ยของผลต่างสัมบูรณ์ ของแท่งทั้ง 96 อัน 

sum += Math.abs(a[i] - b[i]);
return sum / n;   // 0 = เหมือนกันเป๊ะ, ยิ่งมาก = ยิ่งต่าง

diff readout ที่โชว์ Δ last two 23% ก็มาจากตรงนี้ มันดีและเข้าใจง่าย แต่มันมอง รูปร่างแท่งอย่างเดียว — ละเลย features ที่คุณคำนวณไว้แล้ว

ช่องโหว่ที่ตรงกับ mission พอดี สองเสียงอาจมีรูปแท่งคล้ายกัน (barsDistance ต่ำ) แต่ ความรู้สึก ต่างกันชัด — เช่นตัวหนึ่งกระจายกว้าง (bandwidth สูง) อีกตัวเป็นโทนแคบ ถ้าเอา features มาร่วมคิดระยะ signature จะแยกเสียงสัตว์ได้ตรงหูขึ้น

2 · ลองเล่น: เทียบสองเสียง แล้วปรับ “สูตรระยะห่าง”

ปั้นสเปกตรัม A (ส้ม) กับ B (ฟ้า) แล้วดูระยะห่างสามแบบ: bars (แบบที่โค้ดใช้ตอนนี้), features (จาก 4 ตัวเลข), และ blended (ผสมกันตามปุ่มน้ำหนัก) ลองหาคู่ที่ bars บอกว่า “เหมือน” แต่ blended บอกว่า “ต่าง”

ลองสังเกต ตั้ง A กับ B ให้ ตำแหน่งเท่ากัน แต่ ความกว้างต่างกันมาก (A แคบ, B กว้าง):
• bars distance จะค่อนข้างต่ำ (รูปคล้ายกันหลังนอร์มัลไลซ์)
• แต่พอดันน้ำหนัก features ขึ้น blended distance จะพุ่ง — เพราะ bandwidth ต่างกันชัด
นี่คือหลักฐานว่า “เอา features มาร่วม” ทำให้ระยะห่างตรงกับความรู้สึกจริงขึ้น

4 · ลงมือกับโค้ดจริง — ของจริงตาม mission

นี่คืองานที่เอาความรู้ทั้ง 5 บทมาแก้โค้ด Faunaform ให้แยกเสียงสัตว์คมขึ้นจริง:

  1. เปิด js/transforms.js เขียนฟังก์ชันใหม่ featureDistance(fa, fb) ที่เฉลี่ยผลต่างสัมบูรณ์ของ centroid, bandwidth, flatness, rolloff
  2. ทำ signatureDistance(sigA, sigB) ที่ผสม: 0.6 * barsDistance(sigA.bars, sigB.bars) + 0.4 * featureDistance(sigA.features, sigB.features)
  3. ใน signatures.js เปลี่ยน updateDiff ให้เรียก signatureDistance แทน barsDistance ตรง ๆ (ตอนนี้มันส่งแค่ .bars — ส่งทั้ง object เข้าไปแทน)
  4. มี unit test อยู่ใน test/ แล้ว — เพิ่มเทสต์ว่า featureDistance ของ object เดียวกัน = 0 และสมมาตร
  5. จับเสียงจริงเทียบดูว่าค่า Δ ตอนนี้ตรงกับหูคุณขึ้นไหม ปรับน้ำหนัก 0.6/0.4 ตามชอบ
อยากให้ผมช่วยลงมือ ถ้าพร้อมเขียนของจริง บอกผมว่า “ช่วยทำ signatureDistance ให้หน่อย” — ผมจะเขียนโค้ด + เทสต์ ในโปรเจกต์ให้ พร้อมรันเทสต์ให้เห็นว่าผ่าน (นี่คือจุดที่ “เรียน” กลายเป็น “ส่งของ”)
ถามผมได้ทุกเมื่อ — เช่น “น้ำหนัก 0.6/0.4 ควรเป็นเท่าไรดี?”, “อยากให้ระยะห่างคิดเรื่องลีลาเวลา (จากบท 3) ด้วย ต้องเก็บข้อมูลเพิ่มยังไง?”, หรือ “features บางตัวควรถ่วงน้ำหนักมากกว่าตัวอื่นไหม?”

🎓 คุณเรียนครบแกนหลักแล้ว: คลื่น → FFT → spectrogram → features → distance ทั้งหมดผูกกับโค้ด Faunaform จริง

ทิศทางต่อไปที่เป็นไปได้: (ก) เพิ่มโหมด spectrogram เป็น graph form ใหม่ · (ข) ใส่ ลีลาเวลา เข้าไปใน signature เพื่อแยกเสียงหวีดกับเสียงรัว · (ค) เจาะ สเกลเดซิเบล (minDecibels/maxDecibels) ว่ากระทบ feature ยังไง — บอกผมว่าอยากไปทางไหนต่อ
